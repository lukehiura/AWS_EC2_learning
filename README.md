# EC2 + SSH key (Terraform)

Defaults target the **AWS Free Tier (12â€‘month)** style footprint where applicable: **`t2.micro`**, **gp2** root disk **8 GiB** (under the 30 GiB gp2 allocation), single small instance. **Offers differ by account, region, and signup date** â€” confirm against [AWS Free Tier](https://aws.amazon.com/free/).

Terraform project that provisions:

- One **Amazon Linux 2023** **EC2** instance in the **default VPC** (first default subnet).
- An **EC2 key pair** used for SSH.
- A **security group** allowing **SSH (22)** from `var.allowed_ssh_cidr` (default `0.0.0.0/0` â€” tighten for anything real).

## SSH key: two modes

1. **Generated by Terraform (default)**  
   - A **TLS private key** is created; the **public** half is registered as `aws_key_pair`.  
   - After `apply`, read the sensitive output **`private_key_pem`** once and save it locally as `learning-ec2.pem`, then:

     ```bash
     chmod 400 learning-ec2.pem
     ssh -i learning-ec2.pem ec2-user@$(terraform output -raw public_ip)
     ```

   - The private key is also in **Terraform state**. Do not commit state or the PEM file.

2. **Your own key**  
   - Set `ssh_public_key = file("~/.ssh/id_ed25519.pub")` (or paste the `.pub` string) in `terraform.tfvars`.  
   - Use the matching **private** `.pem` / OpenSSH key on your laptop to SSH. No `private_key_pem` output in that mode.

## Prerequisites

- Terraform `>= 1.5`
- AWS credentials (CLI profile, env vars, etc.) with permission to create EC2, VPC read, key pairs, and security groups.

## Quick start

```bash

cp terraform.tfvars.example terraform.tfvars
# Edit terraform.tfvars â€” at least set allowed_ssh_cidr to your /32 if you can.

```

## Outputs

- **`public_ip`** â€” connect here.  
- **`ssh_command`** â€” example using `learning-ec2.pem`.  
- **`private_key_pem`** â€” only when Terraform generated the key; `terraform output -raw private_key_pem > learning-ec2.pem`

## Security notes

- Replace **`0.0.0.0/0`** on SSH with **your IP/32** when possible.
- Prefer **SSM Session Manager** later instead of open SSH + long-lived PEMs.
- Add an **S3 backend** (and locking) before treating this as production.

## GitHub Actions

Same pattern as **AWS_IAM_learning**: workflow [`.github/workflows/terraform.yml`](.github/workflows/terraform.yml) runs **fmt / validate** always, **plan** and **apply** when secret **`AWS_ROLE_ARN`** is set (OIDC, no access keys in GitHub).

1. **GitHub OIDC provider** must already exist in the account (one per account for `token.actions.githubusercontent.com`). If you used **AWS_IAM_learning**, it is already there.
2. Apply this repo locally so Terraform creates role **`github-actions-ec2`** (see `github_actions_ci.tf`) trusted for **`repo:lukehiura/AWS_EC2_learning:*`** (override `github_actions_repository` if your fork/name differs).
3. In **this** GitHub repo: **Settings â†’ Secrets â†’ Actions** â†’ **`AWS_ROLE_ARN`** = `terraform output -raw github_actions_role_arn` after apply.
4. Optional: **Variables** â†’ **`AWS_REGION`**; **Environments** â†’ **`terraform-apply`** for gated manual apply.
